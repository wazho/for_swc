  • iCampus-校園行動歷險
  • 　　使用語言：Android、R、Hadoop
  • 　　參與人員：4人團隊 - 校內同學
  • 　　專案時程：since 2014.01 ~
  • 　　模擬 Google Ingress 之計畫所設計的校園版，為一結合校園社交功能的擴增實境（Augmented Reality）與地理位置服務（Location-Based Service）應用。為了鼓勵學生廣泛探索校園環境並促進附近商家的買氣，用戶必須手持智慧型裝置歷訪校園各角落完成任務以獲取點數，隨後即可利用點數在附近商家消費時換取折扣。站在開發者的角度來看，本程式將會記錄用戶 GPS 及網路定位連線資訊，藉此分析校園人潮流動的變化趨勢，進而提供視覺化（visualized）及客製化（customized）的資訊推薦服務。

  • [個人專題] RO 開心農場 (放棄使用)
  • 　　使用語言：RO script language (rAthena)
  • 　　專案時程：since 2013.04 ~ 2013.10
  • 　　使用 rAthena 模擬器提供的 script，構築在 RO 的環境上所做的開心農場，能夠飼養動物、成長、突變、死亡，耕種植物、成長、收成。起初是因為可以減少不少的美工問題，但後來涉及版權問題、未來擴充性不高，雖然完成度已達 95% 終將此計畫冷凍。 公開版的更新日誌：https://sites.google.com/site/grasscomputernote/now-project/ro_pasture

  • [個人專題 第一階段] 2D Game Engine Script (就是現在網頁)
  • 　　使用語言：Javascript (CreateJS)
  • 　　專案時程：since 2014.04 ~
  • 　　長年累積對 RO 伺服器模擬器的認知，利用 javascript 的複合型引擎 'CreateJS' 打造出只有一些程式邏輯用戶也能夠輕易使用 script 創建出自己 RPG 類型遊戲之引擎。前顯易懂且直覺的指令就能夠生成遊戲物件：
  • 　new Character( x, y, hp, sp ... )
  • 　　創造遊戲角色，進行移動、攻擊、施放技能...。
  • 　new Monster( map, x, y ... )
  • 　　創造怪物角色，自定攻擊玩家的模式，路徑。
  • 　new NPC( map, x, y, event ... )
  • 　　創造非玩家控制角色，與其對話與其他功能。
  • 　....　諸如此類的語法。
  • 　　這其實是方便自己在未來進行其他開發（第二階段的前置，預計用此從新打造出網頁牧場）能夠得心應手，可以說是寫給自己的 API，由於剛開始自學 JS 仍有許多要克服的難處。 上週已放到 Github 上：https://github.com/grass0916/Stardust-Script

  • [個人專題 第二階段] 拍賣場網站系統
  • 　　使用語言：html5、以 jQuery、MySQL 為主
  • 　　專案時程：since 2013.08 ~
